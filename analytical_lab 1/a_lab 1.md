# Исследование AWS сервисов и поиск их доступных аналогов
---
### Amazon Macie
**Amazon Macie** — это сервис обеспечения безопасности данных в Amazon S3, который обнаруживает конфиденциальные данные с помощью машинного обучения и сопоставления шаблонов, а также обеспечивает контроль рисков безопасности данных и автоматическую защиту от этих рисков.
Отличительной особенностью Amazon Macie является автоматическое отслеживание конфиденциальных данных.
**Kaspersky Security для виртуальных и облачных сред** также обеспечивает автоматическую защиту данных и может быть интегрирован с платформами AWS, Microsoft Azure, Yandex.Cloud и Google Cloud Platform через API. Однако, он не предоставляет возможность отслеживания статуса конфиденциальности данных, поэтому не может полноценно претендовать на роль аналога Amazon Macie.
---
### Amazon MSK
**Amazon Managed Streaming for Apache Kafka (MSK)** автоматически выделяет серверы, настраивает кластеры Apache Kafka, заменяет серверы в случае их сбоя, организует исправления и обновления серверов, эффективно проектирует кластеры, обеспечивает надежное хранение и безопасность данных, настраивает мониторинг системы, а также ведет себя гибко при изменении нагрузки. Сервис позволяет обрабатывать потоковые данные в реальном времени.
**Yandex Managed Service for Apache Kafka** помогает разворачивать и поддерживать кластеры серверов Apache Kafka® в инфраструктуре Yandex Cloud и является аналогом Amazon MSK.
---
### Amazon Polly
**Amazon Polly** использует технологии глубокого обучения для синтеза естественной человеческой речи, благодаря чему с его помощью можно преобразовывать текст в речь. Amazon Polly предоставляет API и большое количество голосов и языков для синтеза.
**Yandex SpeechKit**, как и Amazon Polly, позволяет синтезировать речь из текста и работает через HTTP API, а помимо этого поддерживает и обратное преобразование голоса в текст.
Делая вывод, можно сказать, что полное замещение Amazon Polly при помощи Yandex SpeechKit является возможным.
---
### Amazon S3
**Amazon Simple Storage Service (S3)** — это облачное объектное хранилище, которое отличается высокой надежностью, доступностью, производительностью и безопасностью в отрасли, а также практически неограниченной масштабируемостью при очень низких затратах. S3 — это вариант не иерархического хранилища. С точки зрения системы все объекты равнозначны, поэтому в S3-хранилище удобно долго хранить разнородную информацию и быстро получать к ней доступ. Также для работы S3 использует одноименный API.
**Yandex Object Storage** является универсальным автоматически масштабируемым облачным объектным хранилищем в инфраструктуре Yandex Cloud. Совместимость с Amazon S3 API и другими инструментами, совместимыми с Amazon S3, позволяет пользователю без особых проблем мигрировать на российский сервис.
---
### Amazon Personalize
**Amazon Personalize** — это сервис машинного обучения, который помогает разработчикам, не имеющим опыта машинного обучения, быстро создавать и развертывать собственный механизм рекомендаций и легко встраивать в свои приложения сложные возможности персонализации. Amazon Personalize потоково обрабатывает входные данные, подбирает наиболее подходящие алгоритмы и обучает и оптимизирует частную модель персонализации рекомендаций.
На данный момент российских аналогов ему нет.
---
### Amazon ES
**Amazon Elasticsearch Service** — это сервис, который упрощает развертывание, эксплуатацию и масштабирование Elasticsearch в облаке AWS.
Elasticsearch — это популярный поисковой и аналитический движок основанный на Apache Lucene. С его помощью можно также анализировать логи и мониторить приложения в реальном времени.
В связи с изменением лицензии Elasticsearch в 2021 году, Amazon развивает технологию OpenSearch которая сопоставима по функциональности с Elasticsearch.
**Yandex Managed service for Elasticsearch** — является аналогом Amazon Elasticsearch Service, однако компания Elastic сообщила, что с апреля 2024 года доступ к сервису Elasticsearch будет ограничен для пользователей Yandex Cloud, а новые пользователи не смогут создавать кластеры с 20 июля 2023 года.
Поэтому в качестве альтернативы стоит рассмотреть решение **Yandex Managed Service for OpenSearch**, которое упрощает использования OpenSearch для целей эффективного поиска. Преимуществом данного сервиса являются его более выгодные тарифные планы.
